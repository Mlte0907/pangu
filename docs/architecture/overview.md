# 架构概述

## 设计哲学

盘古是智能系统的"大脑"组件，专门负责**记忆**这一核心智能能力。

### 核心原则

1. **不做手脚**：盘古不执行任务、不做决策
2. **不造轮子**：复用成熟方案（ChromaDB / SQLite / FastAPI）
3. **类人模拟**：借鉴认知科学（Miller 7±2、Ebbinghaus 遗忘曲线）
4. **可插拔**：所有后端均可替换
5. **可观测**：内置 Prometheus 指标

## 4 层记忆栈

盘古将记忆分层，模拟人类记忆的工作方式：

```
┌─────────────────────────────────────┐
│ L0 身份（Identity）                 │  ~100 tokens  · 始终加载
│   "我是盘古，专业的记忆系统"          │
└─────────────────────────────────────┘
┌─────────────────────────────────────┐
│ L1 摘要（Summary）                  │  ~500-800 tokens · 始终加载
│   "用户最近 7 天主要关注：..."        │
└─────────────────────────────────────┘
┌─────────────────────────────────────┐
│ L2 按需（On-demand）                │  ~200-500 tokens · 话题触发
│   "上下文：用户在讨论 A 主题"        │
└─────────────────────────────────────┘
┌─────────────────────────────────────┐
│ L3 深度（Deep Search）              │  无限 · 显式查询
│   "完整检索 + 排序 + 上下文组装"     │
└─────────────────────────────────────┘
```

## 记忆宫殿

盘古采用"宫殿隐喻"组织记忆：

```
Palace (宫殿)
├── Wing: tech (空间)
│   ├── Room: python
│   │   ├── Drawer: "用 fastapi 实现 MCP"
│   │   ├── Drawer: "向量化检索的关键点"
│   │   └── Drawer: "..."
│   ├── Room: rust
│   └── Room: go
├── Wing: life
│   ├── Room: health
│   ├── Room: finance
│   └── Room: hobbies
└── Wing: work
    └── ...
```

- **Wing** (空间)：主题域
- **Room** (页面)：子主题
- **Drawer** (抽屉)：具体记忆项

## 知识图谱

实体 + 关系 + 因果链：

```python
kg.add_entity("盘古", "system", {"type": "memory_system"})
kg.add_entity("FastAPI", "framework", {"language": "python"})
kg.add_relation("盘古", "FastAPI", "uses", weight=0.9)
```

支持：
- 实体管理
- 关系管理
- 路径查询（BFS）
- 因果链追溯
- 多视角视图

## 工作记忆

模拟 Miller 7±2 短期记忆：

- 容量：默认 40 槽位（7 × 3 安全系数）
- 激活度衰减
- 情感保护（高情感值不易被驱逐）
- 紧迫度保护（高紧迫度优先保留）
- 自动检查点持久化

## 长时记忆

借鉴 Ebbinghaus 遗忘曲线：

```
importance(t) = importance(0) × decay_base^t × touch_boost × night_factor
```

- 时间衰减
- 访问加成（touch）
- 夜间加速衰减
- 衰减地板（不无限趋近于 0）

## 伏羲移植模块

盘古从伏羲（fuxi）系统移植了 16 个核心模块：

| 模块 | 作用 |
|:---|:---|
| FTS5 搜索 | FTS5 + 向量 RRF 融合 |
| 全息记忆 | 5 维语义投影 |
| 记忆法官 | LLM A/B/C 价值判断 |
| 自适应参数 | 状态驱动参数调整 |
| 工作记忆 | Miller 7±2 短期记忆 |
| 脱敏 | 3 级 PII 检测 |
| 再巩固 | 衰减记忆刷新 |
| 共鸣 | 跨 Wing 语义关联 |
| 知识蒸馏 | 结构化知识卡片 |
| 向量索引 | 大规模 ANN 搜索 |
| 注意力 | 5 策略 + A/B 测试 |
| 增强评估 | 6 种 LLM 裁决 |
| 流式索引 | 增量 + WAL |
| 验证循环 | 6 阶段质量门控 |
| 差分隐私 | ε-隐私保护 |

## 数据流

```
用户输入 → MCP 调用 → 记忆处理 → 持久化
                            ↓
                       知识图谱更新
                            ↓
                       Wiki 自动生成
                            ↓
                       衰减 / 巩固调度
```
